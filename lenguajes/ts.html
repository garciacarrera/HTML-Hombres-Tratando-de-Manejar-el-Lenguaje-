TypeScript

Características

    Tipo de usos

  1. Su uso principal es en desarrollo web front-end y back-end. Es muy utilizado para aplicaciones web front-end grandes y complejas, mejorando la mantenibilidad y escalabilidad del código. Del lado del back-end se acompaña bastante con Node.js y frameworks como NestJS, usado para el desarrollo del lado del servidor de apps web. Es ideal para proyectos grandes y colaborativos donde la claridad y la detección temprana de errores son cruciales.

    2. Paradigmas

  2. Su paradigma es orientado a objetos, usando clases y objetos para modelar entidades del mundo real, tal como JavaScript. Es funcional en menor medida, permitiendo el uso de funciones de orden superior y expresiones para crear código más declarativo y reutilizable. También es compatible con programación genérica, donde se definen funciones y estructuras de datos que trabajan con tipos genéricos, mejorando la flexibilidad del código y su reutilización.

    Historia

  • Desarrollado en 2012 por la empresa Microsoft, creado como un tipo de superconjunto de JavaScript para mejorar la escalabilidad y seguridad de aplicaciones web grandes. Diseñado para agregar tipado opcional a JavaScript, permitiendo a los desarrolladores escribir un código mucho más confiable y con menos errores. Ha sido adoptado por un gran número de empresas y proyectos debido a sus características y compatibilidad con JavaScript.

    • Tipado

  • Es de tipado Estático opcional, permitiendo a los desarrolladores agregar opcionalmente tipos a sus variables y funciones. Verifica los tipos en tiempo de compilación para detectar errores potenciales antes de ejecutar el código. Ofrece compatibilidad con todos los navegadores web porque se compila a JavaScript estándar.

    • Nivel de lenguaje

  • Es de alto nivel, abstrae los detalles de la arquitectura de una computadora permitiendo a los programadores centrarse en la lógica del programa. Su sintaxis está basada en JavaScript, con la adición de anotaciones de tipo.
