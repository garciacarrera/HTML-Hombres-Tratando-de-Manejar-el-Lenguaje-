Go

Características

    Tipo de usos

  1. Go es muy popular para construir sistemas y herramientas de infraestructura, como servidores, sistemas distribuidos y software de red debido a su eficiencia y simplicidad.

    2. Paradigmas

  2. Procedimental: Go se basa en el enfoque procedural, donde el programa se organiza en procedimientos o funciones que operan sobre datos. Orientado a objetos (limitado): Aunque Go no tiene clases ni herencia tradicional como otros lenguajes orientados a objetos, utiliza estructuras (structs) y métodos para ofrecer una forma de programación orientada a objetos. La composición de tipos y las interfaces son las herramientas principales para lograr una forma de polimorfismo.

    Historia

  • Go, también conocido como Golang o Google Go, nace en el año 2007 y fue desarrollado por los ingenieros Rob Pike, Ken Thompson y Robert Griesemer, quienes buscaban que este fuera un lenguaje escalable como lo es C++ y Java, sin embargo su lanzamiento fue llevado a cabo por Google en el año 2009.

    • Tipado

  • En Go, los tipos de las variables se determinan en tiempo de compilación, lo que significa que el tipo de una variable no puede cambiar durante la ejecución del programa. Esto ayuda a detectar errores de tipo en tiempo de compilación, aumentando la robustez del código.

    • Nivel de lenguaje

  • Go combina características de lenguajes de alto nivel (como la simplicidad y la facilidad de uso) con algunas características de lenguajes de bajo nivel (como el control sobre la memoria). Esto hace que Go sea adecuado tanto para desarrollo de aplicaciones de alto nivel como para sistemas que requieren eficiencia y control.
